心地よいを考えてみる(カルーセルUI)

パッと思いつくストレスたち

  • 重すぎるサイト　zara ロエベ ハパクリスティ
  • 挙動の始まりがモッサリしている
  • 遷移が遅い（何秒で遅いと感じるのか）
  • 欲しい情報が見つからない
  • ボタンを押したが遷移が遅く、ボタンが押下できているかどうか微妙
  • モーダルを開き、閉じると結構なページまで戻ってしまう
  • やたらとポップアップが出る
  • カルーセルで情報が読めない　某飲料top

気になる統計

  • ページ遷移に何秒かかると人は遅いとストレスに感じるのか
  • 動く情報はどの程度人が認知することができるのか

結論こうらしい

  • 遷移に関しては3秒以内表示速度と直帰率(google英文)
  • 表示に3秒かかると、直帰の確率は32%増加,表示に5秒かかると、直帰の確率は90%増加表示に6秒かかると、直帰の確率は106%増加表示に10秒かかると、直帰の確率は123%増加
  • 1秒で10文字以内を目指す
  • 文字の大きさは16pxが好ましい
  • バナーの数は3-5が好ましい→短期記憶で処理できる量
  • →バナーの数は3-5までが好ましい？
  • 文字の大きさは16pxが良い

(引用)人の記憶のうち、短期的に保持される記憶のことを「短期記憶」といいます。ミズーリ大学の心理学教授、ネルソン・コーワン氏の研究によると、短期記憶で処理できる情報量は4±1個程度だそうです。これに従えば、4±1を越える情報を探させたり、選ばせたりすると、短期記憶が使えず、脳の負荷が高まると考えられます。逆に、4±1より少なくすれば、脳への負荷が減り、認知が容易になり、結果的に使いやすく感じる可能性が高まるともいえます。

(引用)一般に人の読みの能力に関しては，適切な条件下で1回の視線の停留で読めるのは漢字かな混じり文で3.5文字程度である。停留は平均して300m秒程度必 要でなおかつ眼球の移動（跳躍運動）に数10m秒必要なことから，静止した文章でも1秒間に読める文字数は10文字程度となる。実際には，変化したり移動 したりする文字では，読みの速度がさらに低下すると考えられるので，1秒間で5文字程度と考えることができる。（『JIS X 8341-3:2004 「高齢者・障害者等配慮設計指針－情報通信における機器，ソフトウェア及びサービス－ 第３部：ウェブコンテンツ」 技術解説　第1.1版　委員会ワーキングドラフト（7月22日版）』より）
